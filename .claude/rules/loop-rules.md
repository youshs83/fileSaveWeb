## 필수 규칙

1. 테스트 없이 코드를 작성하지 않는다
   - 항상 테스트 먼저 작성 후 코드를 구현한다

2. 실패 루프 제한
   - 동일 테스트가 5회 연속 실패하면 루프를 중단하고 사용자에게 보고한다
   - 무한 루프 방지를 위한 안전장치

3. DB 접속정보는 .env에만 저장한다
   - 코드에 하드코딩 금지
   - conf/.env 파일 사용, 커밋 금지

4. 한 사이클에 한 기능만 처리한다
   - 로그인 API와 파일 업로드를 동시에 구현하지 않는다
   - 순서대로 완성 후 다음으로 이동

5. 에러 수정 시 기존 통과 테스트를 깨뜨리지 않는다
   - 수정 후 전체 테스트를 재실행하여 회귀 확인

6. 로그 기록
   - 각 사이클 시작/종료/실패를 logs/loop-activity.log에 기록한다
